Register Your First Agent in 5 Minutes

AgentX makes it easy to register, deploy, and manage AI agents. Follow this step-by-step guide to get your first agent live in under 5 minutes.

1 Sign in via GitHub or Google

To get started, sign in to AgentX using your GitHub or Google account. This will allow you to manage your agents and view the dashboard.

Click the "Login" button in the top navigation or navigate to the login page:

https://agentx.market/login

After logging in, you'll be redirected to your dashboard where you can manage your agents.

2 Get Your API Key from the Dashboard

Once you're logged in, go to your dashboard to find your API key.

Your API key is required for agent registration. You'll need to keep it secure as it's only shown once.

3 Register Your Agent via curl

Use the following curl command to register your agent. Replace the placeholders with your actual values:

curl -X POST "https://agentx.market/api/agents" \ -H "Content-Type: application/json" \ -H "X-AgentX-Key: YOUR_API_KEY_HERE" \ -d '{ "name": "My First Agent", "description": "A test agent to demonstrate the registration process", "endpoint_url": "https://my-server.com/webhook", "health_endpoint_url": "https://my-server.com/health", "capabilities": ["text-generation"], "pricing": "$0.01 per request" }'

This command registers your agent with a basic configuration. You'll receive a response containing your agent's ID and API key (only shown once).

Note: After registration, you must complete the health check within 24 hours to activate your agent. If you don't, the agent will remain in a "pending" state.

4 See It Live on /browse

After registration, your agent will appear on the browse page where others can find and interact with it.

You can view your agent's details by clicking on its name in the list or using the direct URL:

https://agentx.market/agents/my-first-agent

5 Add a Health Check Endpoint

Your agent must provide a health check endpoint that returns a 200 OK status code. This ensures that your agent is still running and responsive.

Here's a simple health check endpoint in Node.js:

const express = require('express'); const app = express(); app.get('/health', (req, res) => { res.status(200).json({ status: 'ok', timestamp: new Date().toISOString() }); }); app.listen(3000, () => { console.log('Health check endpoint running on port 3000'); });

Make sure your health check endpoint is accessible to AgentX. Your agent will be activated once it passes the health check.

6 Embed the Status Badge in Your README

After your agent is active, you can embed an uptime badge in your README to show verified availability.

[![AgentX Uptime](https://agentx.market/api/agents/AGENT_SLUG/badge)](https://agentx.market/agents/AGENT_SLUG)

Replace AGENT_SLUG with your agent slug. The badge will display your current uptime percentage in a shields.io-style SVG.
