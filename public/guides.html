Slack

Send AgentX events into a Slack channel

Use an incoming webhook when you want deploy alerts, health changes, or operator actions to land in a single channel with minimal setup.

  1. 1. Create a Slack incoming webhookIn Slack, add the Incoming Webhooks app and generate a webhook URL for the target channel.
  2. 2. Store the webhook securelySave it as SLACK_WEBHOOK_URL in your deployment environment.
  3. 3. Post a structured payloadSend JSON with a short title, status, and deep link back to the relevant AgentX page.
curl example Slack webhook 
curl -X POST "$SLACK_WEBHOOK_URL" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "AgentX alert",
    "blocks": [
      {
        "type": "section",
        "text": {
          "type": "mrkdwn",
          "text": "*Health check passed*\nAgent `support-router` is healthy again."
        }
      },
      {
        "type": "context",
        "elements": [
          {
            "type": "mrkdwn",
            "text": ""
          }
        ]
      }
    ]
  }'
Use Slack blocks when you need readable incident updates; keep the payload flat and link operators back to /status or the relevant listing.
Discord

Broadcast updates with a Discord webhook

Discord webhooks work well for community channels, internal ops rooms, or lightweight status broadcasts without maintaining a bot.

  1. 1. Create a channel webhookOpen channel settings in Discord, create a webhook, and copy the generated URL.
  2. 2. Save it as an environment variableUse DISCORD_WEBHOOK_URL in your app or worker.
  3. 3. Include embeds for contextEmbeds let you show status, actor, and a button-style link in a compact message.
JSON payload Discord webhook 
{
  "username": "AgentX Monitor",
  "content": "New marketplace event",
  "embeds": [
    {
      "title": "Listing verified",
      "description": "The `support-router` listing is now marked verified.",
      "color": 2262786,
      "fields": [
        { "name": "Operator", "value": "marco", "inline": true },
        { "name": "Environment", "value": "production", "inline": true }
      ],
      "url": "https://agentx.market/browse"
    }
  ]
}
Discord expects a plain webhook POST. If you need retries or fan-out, place a small worker between AgentX events and the webhook endpoint.
GitHub Actions

Call AgentX from CI after deploys or checks

GitHub Actions is the fastest path for reporting deploy results, syncing listing metadata, or pinging an internal endpoint after tests pass.

  1. 1. Add a repository secretCreate AGENTX_API_KEY in GitHub repository or organization secrets.
  2. 2. Trigger after your build or deployRun the AgentX step only after the workflow reaches a stable state.
  3. 3. Send a signed requestUse the API key in the X-AgentX-Key header and keep payloads explicit.
workflow yaml .github/workflows/notify-agentx.yml 
name: Notify AgentX

on:
  workflow_dispatch:
  push:
    branches: [main]

jobs:
  notify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Report deploy to AgentX
        run: |
          curl -X POST "https://agentx.market/api/contact" \
            -H "Content-Type: application/json" \
            -H "X-AgentX-Key: ${{ secrets.AGENTX_API_KEY }}" \
            -d '{
              "name": "CI Bot",
              "email": "ops@example.com",
              "subject": "Deploy completed",
              "message": "Production deploy finished from GitHub Actions."
            }'
If you later add a dedicated automation endpoint, keep the same secret pattern and tighten the payload contract instead of overloading free-form text fields.
Zapier

Bridge AgentX into no-code workflows

Zapier is the lowest-friction option when a non-engineering team needs to route form submissions, alerts, or listing events into downstream tools.

  1. 1. Start with Webhooks by ZapierChoose a Catch Hook trigger if AgentX will send data into Zapier, or POST if Zapier will call out.
  2. 2. Map the core fieldsKeep a small schema like agent name, event type, environment, and URL.
  3. 3. Fan out to business toolsFrom Zapier, route to Google Sheets, Slack, email, CRM, or an internal webhook.
javascript fetch Catch Hook source 
await fetch("https://hooks.zapier.com/hooks/catch/123456/agentx/", {
  method: "POST",
  headers: {
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    event: "new_lead",
    agent: "support-router",
    source: "agentx_marketplace",
    operator: "marco",
    url: "https://agentx.market/contact",
    created_at: new Date().toISOString()
  })
});
Zapier works best when the payload is boring: predictable field names, ISO timestamps, and stable event strings make downstream steps easier to maintain.
